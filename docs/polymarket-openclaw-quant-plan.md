# Polymarket + OpenClaw 赛马量化系统落地蓝图

更新时间：2026-03-04

## 1. 目标定义

构建一个在 Polymarket 运行的“赛马式多策略量化系统”，由本地 AI 模型 OpenClaw 参与信号生成与策略选择，实现：

1. 模拟盘阶段：无真实下单，完整复现撮合、手续费、滑点、仓位与风控。
2. 小资金实盘阶段：受限额度真实下单，严格风控与回滚机制。
3. 稳定实盘阶段：可持续运行，具备监控、告警、审计与复盘能力。

## 2. 关键约束（必须先确认）

1. 账户与地区合规：官方帮助中心有“受限制地区”说明，需先确认你当前运行环境与账户可合法实盘。
2. API 鉴权：Polymarket 使用 API Key + Secret + Passphrase，并要求签名头。
3. 官方 CLI 角色：`polymarket-cli` 可作为“参考实现/运维工具”，核心交易引擎建议独立实现，避免 CLI 变更影响生产系统。
4. OpenClaw 本地部署：推理延迟、并发、显存/内存上限会直接影响下单时机。

## 3. 系统架构（建议）

```text
[Market Data Ingest]
  - CLOB/市场数据
  - 事件与盘口快照
        |
        v

  - 事件特征（时间、流动性、波动）
        |
        v
[Strategy Arena (赛马)]
  - S1 做市偏被动
  - S2 趋势跟随
  - S3 均值回归
  - S4 事件驱动
        |
        v
[OpenClaw Meta-Policy]
  - 选策略/配权重/调风险参数
        |
        +----------------+
        |                |
        v                v
[Paper Executor]    [Live Executor]
  - 模拟撮合           - 真实下单
  - 成本建模           - 订单状态追踪
        |                |

                v

                |

```

## 4. 分阶段里程碑（8-10 周）

### M0（第 1 周）：工程底座

1. 建立 monorepo：`apps/trader`、`libs/core`、`libs/connectors`、`infra`。
2. 接入配置管理与 secrets（本地 `.env` + 密钥分层）。
3. 打通“只读链路”：拉取市场数据 -> 存储 -> 可视化检查。

验收：
- 能稳定抓取目标市场（如赛马相关或二元事件市场）数据 24h。

### M1（第 2-3 周）：模拟盘执行器

1. 实现订单模型：限价/市价、部分成交、撤单。
2. 实现成本模型：手续费 + 滑点 + 延迟。
3. 实现资金账本与持仓估值。

验收：
- 给定历史数据回放，能够输出净值曲线、回撤、胜率、PnL 分解。

### M2（第 4-5 周）：赛马策略框架

1. 策略接口统一：`on_tick -> signal -> order_intent`。
2. 并行运行 3-5 套策略，记录各策略实时分数。
3. 策略切换机制：Top-K + 冷却时间 + 交易成本惩罚。

验收：
- 同一时段可比较策略表现并自动切换，不出现频繁抖动切换。

### M3（第 6 周）：OpenClaw Meta-Policy

1. 用 OpenClaw 读取策略状态摘要（不是原始全量 tick，控制延迟）。
2. 输出：策略权重、风险预算、禁交易标志。
3. 增加“AI 守门”：若输出异常，回退到规则策略。

验收：
- OpenClaw 参与后，模拟盘风险调整后收益优于纯规则基线（至少一个稳定窗口）。

### M4（第 7-8 周）：小资金实盘

1. 实盘开关、双确认、每日上限。
2. 下单失败重试、幂等键、断线恢复。
3. 监控告警（订单失败率、延迟、净敞口超限）。

验收：
- 连续运行 7 天无严重风控事故，且日志可完整审计。

### M5（第 9-10 周）：稳态运营

1. 每日自动复盘报告。
2. 参数自动调优（仅在模拟盘/影子盘先验证）。
3. 灰度放量机制。

验收：
- 实盘稳定运行并满足你定义的收益/回撤目标。

## 5. 风控红线（实盘前必须实现）

1. 单市场最大风险敞口（例如账户净值的 5%-10%）。
2. 单日最大亏损阈值（例如 2%-3%）触发停机。
3. 最大回撤阈值触发降杠杆/只平不开。
4. 异常行情保护：盘口深度不足、价差异常扩大时禁开新仓。
5. AI 输出白名单：只允许有限动作集合，不允许直接“任意下单”。

## 6. 建议技术栈

1. 交易主进程：Python（快迭代）或 Rust（稳定高性能），先 Python 后 Rust 重构。
2. 策略/回测：Python + Pandas/Polars。
3. 数据与状态：PostgreSQL + Redis。
4. 可观测性：Prometheus + Grafana + Loki。
5. 本地模型服务：OpenClaw 以 HTTP/gRPC 本地推理服务暴露。

## 7. 第一阶段（本周）执行清单

1. 定义交易对象范围：先选 1-2 类市场，不要全市场并行。
2. 完成读行情与账户只读 API 封装。
3. 搭建模拟盘执行器最小可运行版本（MVP）。
4. 输出第一版日报模板：净值、持仓、交易、风控事件。

## 8. 立即开始的落地顺序（Step-by-step）

Step 1：项目脚手架 + 配置模板 + 数据抓取器（只读）
Step 2：模拟盘撮合与账本
Step 3：3 个基线策略接入（无 AI）
Step 4：OpenClaw 只做“策略配权”
Step 5：小资金实盘灰度
