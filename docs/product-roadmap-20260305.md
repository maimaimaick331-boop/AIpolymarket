# 产品化剩余规划（2026-03-05）

目标：把当前系统从“可运行”推进到“可上线试运营”的产品状态。

## 里程碑 M1（本轮已完成）

1. 主流程串联  
   - 模型接入 -> 策略生成任务 -> 模拟盘启动联动 -> 数据可见。
2. 任务可观测  
   - 异步 job、进度、事件、历史回看。
3. 页面降噪  
   - 高级配置折叠、默认显示主路径。
4. 自动化冒烟  
   - 脚本化验证核心链路。

## 里程碑 M2（下一步，必须完成）

1. 策略工坊增强  
   - 策略模板库（保守/平衡/激进）。
   - prompt 历史与复用。
2. 模拟验证增强  
   - 单策略与多策略组合回测视图。
   - 分 token 胜率、手续费占比、滑点估算。
3. 结果判定门禁  
   - 可配置晋级阈值（最小样本、最大回撤、最低胜率、最小净收益）。

验收标准：
- 同一个策略在页面中可看到：参数、事件、订单、成交、PnL、晋级判定结果。

## 里程碑 M3（实盘灰度）

1. 实盘控制台  
   - 小额灰度开关、手动急停、策略黑名单。
2. 风险守护  
   - 日内亏损上限、单 token 暴露上限、连续亏损熔断。
3. 审计追踪  
   - 每笔下单保留 request/response 与策略上下文。

验收标准：
- 灰度期间任何时刻可一键停单，并可回溯近 N 笔实盘决策链路。

## 里程碑 M4（运维与可靠性）

1. 服务可靠性  
   - 后台守护进程、重启恢复、状态告警。
2. 数据可靠性  
   - 关键状态快照、日志切割、异常追踪。
3. 发布规范  
   - 版本号、变更记录、回滚步骤。

验收标准：
- 连续运行 24h 无崩溃，关键数据不丢失，可恢复。
